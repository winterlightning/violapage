1 Getting Startedtop

1.1 Download Package Contents


Once you have downloaded and unzipped the Mentor package you will be faced with a list of files and folders. An explanation of these is as follows. Any item not mentioned here should be ignored for the moment.

Folders
cssThis folder contains all of the css for the template
docsThis is the help documentation. To view simply open index.html found inside this folder
imgAll images for the template are found here. The images are split further into 2 sub folders Content & Skin. The skin folder contains all of the images required for the template's skin, while the content folder contains all of the dummy content for the template.
jsThis folder holds all javascript sources for the template and its plugins
phpThe contact form and newsletter form php scripts can be found here
psdThe psd files required to change an image based element of the skins are stored here
Files
favicon.jpgThis is the little bookmark icon, to be replaced with your own icon
*.htmlThose are all Mentor example pages shown in Themeforest preview
uncompressed.htmlThis file is a copy of the index.html file, but without any minification/compression of the css and javascript sources for the template. To replace any of the minified CSS or JavaScript in the main template html files, simply copy the import statements from this file and paste them into the same locations in any of the .html files.

1.2 Setting Up Your Site


The download package contains a whole example of a website. You are free to use it as your website or simply just as a starting point. All pages work independently, so you can pick and choose what pages to use and what to omit.

1.3 Select a Skins


The Mentor template comes with 3 pre-made skins; Green, Blue & Orange. The default skin is the Green skin. Skins can be changed in the template by inserting the following import statement into the head section of each HTML page in the template. The code below shows the blue skin being selected, but to select the orange skin instead, simply use skin_orange in place of skin_blue.
/*inserting the blue skin */
<link href="css/skin_blue.css" rel="stylesheet"/>
If no skin CSS stylesheet is inserted, the template's styles will default to the Green Skin.
When inserting the skin import statement, make sure that it comes after/below the import statement of the main CSS file for the template style.css and the import statement of the minified CSS, if the minified CSS is used instead.

3 Twitter Bootstrap Widgetstop

Twitter bootstrap framework includes several widgets like tooltips, tabs, accordions and so on. The Mentor template's CSS styles these widgets so they match the template's design and also provides helper methods so they can be easily and added anywhere in a page.



3.1 Tooltips

To add a tooltip, use an a tag and set its data-rel attribute to tooltip
<a href="#" title="Get RSS Feed" data-rel="tooltip" data-position="bottom" >Text</a>
Parameter / Value Explanation
data-relWhen set to "tooltip", enables the widget
titleSets tooltip content
data-positionTooltip position: top, bottom, left or right

3.2 Popovers

To add a popover, use an a tag and set its data-rel attribute to popover
<a href="#" data-rel="popover"  title="Title" data-content="Content">hover for popover</a>
Parameter / Value Explanation
data-relWhen set to "popover", enables the widget
titleSets popover title
data-contentSets popover content

3.3 Tabs

Tabs can be added using the following markup
<ul class="nav nav-tabs">
    <li class="active"><a href="#home" data-toggle="tab">Home</a></li>
    <li><a href="#profile" data-toggle="tab">Profile</a></li>
    <li><a href="#messages" data-toggle="tab">Messages</a></li>
    <li><a href="#settings" data-toggle="tab">Settings</a></li>
</ul>


<div class="tab-content">
    <div class="tab-pane active" id="home">...</div>
    <div class="tab-pane" id="profile">...</div>
    <div class="tab-pane" id="messages">...</div>
    <div class="tab-pane" id="settings">...</div>
</div>
The ul section defines tab headers, active class selects the default opened tab. Each link must have data-toggle attribute set to tab and href pointing to a unique anchor. Tab content goes into a div placed right after the ul, each unique anchor sets in there must have a related div placed in content section where div id = anchor.

3.4 Accordions

Accordions can be added using the following markup
<div id="accordion2" class="accordion">
    <div class="accordion-group">
        <div class="accordion-heading">

        </div>
        <div id="collapseOne" class="accordion-body collapse">
            <div class="accordion-inner">Lorem ipsum dolor sit amet dapibus id</div>
        </div>
    </div>
    <div class="accordion-group">
        <div class="accordion-heading">
            <a class="accordion-toggle" href="#collapseTwo" data-parent="#accordion2" data-toggle="collapse">Heading 02 </a>
        </div>
        <div id="collapseTwo" class="accordion-body collapse">
            <div class="accordion-inner">Lorem ipsum dolor sit amet nullam mollis </div>
        </div>
    </div>
    <div class="accordion-group">
        <div class="accordion-heading">
            <a class="accordion-toggle" href="#collapseThree" data-parent="#accordion2" data-toggle="collapse">Heading 03 </a>
        </div>
        <div id="collapseThree" class="accordion-body in collapse">
            <div class="accordion-inner">Lorem ipsum dolor sit amet nisi neque</div>

Like in the previous tabs example, for each accordion entry we have 2 sections: accordion-heading defines the title and includes a link to the related accordion-body which holds the content. The anchor set in heading must match the id of the content-section. Additionally, each accordion entry must reference its parent container id in the data-parent attribute.

4.3 Twitter Widget


The twitter widget fetches the latest tweets from a given username account and displays them. To use the widget, the following markup is required:
<div class="twitter" data-username="pixelentity" data-count="4"></div>
Parameter / Value Explanation
data-usernametwitter account username from which tweets will be loaded
data-countthe number of tweets which will be displayed
The tweets are cached, this means once the widget got the data from twitter, it stores it in browsers for about 15 minutes.
This makes navigation faster because data doesn't need to be reloaded on each page reload. So if you're using your account and have just published a new tweet, don't worry if you don't see it immediately on the page as it will only be shown once the current tweet cache expires.

Contact Form Setup
To set up the contact form you need to make a couple of simple changes to the contact.php file which is located in the php folder. Open this file with any standard code editor and you will see the following code in the first few lines of the file.

// CUSTOMIZATION START


$to = "youremail@yourdomain.com";  // insert your contact email address here
$subject = "Contact Form Subject Message";  // this will be the subject line 


// CUSTOMIZATION END


The above 2 lines are the only things which need to be changed in this php file. Simply insert your own email address into the first line. This will be the address to which all submitted contact form messages are sent. The phrase entered in the second line will be the subject line of the emails received from this contact form. This allows you to enter a keyword which you can then use in your email client to automatically filter these emails into a separate folder for example.
The contact form in the Mentor template requires PHP files to function. This means that your files will need to be on a server before the form will work properly. Additionally your server will need to support PHP, which is not the case with Microsoft .ASP servers. For any other troubleshooting regarding the contact form, see the FAQ section on our Support Forum here: http://support.pixelentity.com/forums/1-Support

7 Inline Videostop

In this section we will look at how to insert\embed videos into the main content areas of normal pages. Mentor supports the playback of local video sources as well as online video sources such as YouTube, Vimeo & Vid.ly. Embedding a video is extremely easy, you just need to use an a tag with a class of peVideo and the href attribute set to the video's source (if local) or the video's full url (if online). The following are 3 examples of the required markup for online videos:


7.1 Online Videos

<a class="peVideo" href="http://www.youtube.com/watch?v=AFA-rOls8YA"></a>
<a class="peVideo" href="http://vimeo.com/39251390"></a>
<a class="peVideo" href="http://vid.ly/5u4h3e"></a>
All the above examples will automatically fetch the video cover images from the source site and display a clickable play icon so the user knows it is a video.
As mentioned above Local videos are also supported, you will need to provide multiple encodings / formats of the video if it is to be supported by all browsers. The following is an example of the markup required for local video embedding.


7.2 Local Videos

<a class="peVideo" href="videos/video.mp4" data-formats="webm,ogv" data-poster="videos/poster.jpg"></a>
In this case, the cover image location must be provided manually using the data-poster tag.

Local Video Parameters / Attributes Description:
data-formats define the additional formats you need to encode your video in order to use html5 video playback in all browsers. This basically means you should have on your server:

intro.mp4
intro.webm
intro.ogv
data-posteris the video cover image shown before the user clicks play
When uploading these various versions of the same video each copy must have the exact same filename, just with a different file extension.

9 The Responsive Lightboxtop

Mentor features "flare": a custom responsive, touch enabled, mobile optimized lightbox which can be used to display single images/videos or entire galleries. Items can be browsed using buttons, swipe gestures on mobile and mousewheel/keyboard on desktops. It uses hardware accelerated css transforms and fallbacks to standard jquery animation with in older browsers. It supports youtube, vimeo, vid.ly and local videos playback.


9.1 Single image

<a
     data-target="flare"
     href="image.jpg"
     data-flare-scale="fit"
     >


    Single Image</a>
data-targetWhen sets to "flare", activates the lightbox when user clicks on the item
data-flare-scaleControls the image scaling, possible values are

"fit" - Fits the window: no cropping, always show entire image (default mode)
"fill" - Fills the window: horizontal/vertical cropping may occur when image and window aspect ratios are different (almost always)
"fitmax" - Same as "fit" but upscales the image if smaller than window
"fillmax" - Same as "fill" but upscales the image if smaller than window

11.1 The Fluid Grid


Mentor is a fully responsive template, this means that no matter what device a visitor is viewing your website on, the contents and the site layout will adjust to suit that devices default screen resolution. The default layout width of the mentor template is based on a maximum width of 940px. This will automatically remain centered on large resolution screens.

The fluid grid system uses percents for column widths instead of fixed pixels. The grid is a 12 column base CSS grid. A new row in the grid is denoted by the row-fluid class name while the width of a column or columns are denoted by a span# class name, where # represents a number of columns to span out of the base 12. So for example if you would like a div element to be half the width of its container you would use the following:
<div class="span6">...</div>
So following on from the above, the markup required for 2 column layout each column being half the size of its container, would be as follows:
<div class="row-fluid">
 <div class="span6">...</div>
 <div class="span6">...</div>
</div>
The total span number of columns inside one row-fluid element is 12. Also if the containing row-fluid element itself is wrapped by a div which only has a span of 6 columns, the max total span number of the columns is still 12. The reason for this is because the span number doesn't represent a fixed number of pixels but a percentage or ratio width. Therefore a wrapping div with span6 which contains a nested div with span6 would simply mean that the nested column would be 50% of its containing element, which in this case is a div of span6.
Here is an example of the markup required for nesting columns in the fluid grid:
<div class="row-fluid">
  <div class="span12">
   Level 1 of column

   <div class="row-fluid">
    <div class="span6">Level 2</div>
    <div class="span6">Level 2</div>
   </div>
  </div>
 </div>

11.2 Responsive Layout


Mentor CSS works by detecting the resolution of the device which is viewing the website and then based on its size it serves up CSS styles specific to that resolution. The following table represents the supported devices their resolutions and the resulting column widths/layout.

SmartphonesResolution: 480px and below. Column Width: 280px
Smartphones to TabletsResolution: 481px to 767px. Column Width: 420px
Portrait TabletsResolution: 768px to 979px. Column Width: 42px
Desktop/LaptopResolution: 980px and above. Column Width: 60px
The CSS monitors the current device resolution using a technique called Media Queries. This is done inside the CSS stylesheet and looks like the following:
// Landscape phones and down
    @media (max-width: 480px) { ... }


    // Landscape phone to portrait tablet
    @media (max-width: 767px) { ... }


    // Portrait tablet to landscape and desktop
    @media (min-width: 768px) and (max-width: 979px) { ... }


    // Large desktop
    @media (min-width: 1200px) { ... }
All of the styles specific to a certain resolution are nested inside that specific media query (where the {...} occurs above. This means that if you wish to edit the CSS styles for a specific device layout, you will need to do so by locating the correct media query and editing only the CSS nested within it.

12 Mentor Custom CSStop

The main stylesheets for this template can be found in the css folder in the root package. The main stylesheet is named style.css. See the next section for instruction on how to find this information.


12.1 Locating The CSS Style Rules For A Specific Page Element


In the past we have had many support questions from buyers asking how to locate the CSS style statements which control the CSS for a specific element in the theme. So in response we have written this short section describing how to locate the correct CSS file and the exact line of code. Basically there are several options but here we will outline 2 of the easiest.

1- Use FireBug Firefox Add-on
This is a very handy little add on for the Firefox browser which can be found here https://addons.mozilla.org/en-US/firefox/addon/firebug/. Once installed, using the right mouse button whilst hovering over any item in the browser's main page area will bring up a context menu with the option Inspect Element shown at the bottom of the list. Clicking this option will open a window at the bottom of your browser. As standard this window will show the HTML structure on the left and the CSS styles on the right as they specifically pertain to the item over which you were hovering when you right clicked in the beginning. On the right side CSS panel the rules are in a scrollable list, the upper most rules being those greatest in hierarchy when it comes to applying the resulting style information to the item. On the right hand side of this panel inline with each individual CSS selector you will see a filename and a line reference number. With this information you can find the exact file and the exact line on which to change this element's CSS.

2- Google Chrome Browser
Like with Firebug the Google Chrome browser has similar element inspecting capabilities but in this case they are baked right into the browser itself and so no separate add-on is required. As described above the element selecting and inspecting process is the exact same, with the CSS stylesheet and exact line reference being easily accessible in the bottom right CSS styles panel. if you are a Chrome user, perhaps this method is more convenient for you.
Most modern browsers currently have a similar element inspector. Opera & Safari have a native version which can usually be accessed using a right click like Chrome. IE9 also has something similar but it need to be turned on first by clicking F12 key. To select element you then need to use the selector tool (arrow symbol).

12.2 Modifying The template's CSS (Simple Version)


The following method can be used to modify the template's CSS styles. This method is only recommended when the CSS modifications are light and few in number, as overuse of internal CSS is contrary to best practices of CSS. The advantages of this method is that you do not need to touch the compressed stylesheets (see next section). To use this simple method, the following style tags need to be placed inside the head section of each HTML page in which you required the changes to appear. Then inside of these style tags, CSS selectors can be placed as normal.
<!-- modified CSS -->
    <style>
.taglineWrap{
    background:#eee;
    border-bottom:1px solid #ddd;
}
    </style>
Shown above is an example CSS selector to change the color of the Tagline element's background. To find out what kind of selectors are required to target a specific element in the template, refer to the previous section.
NOTE: Sometimes when using the above method for CSS modification, if a CSS style modification does not result in the expected style change even though the CSS selector syntax is correct, you may need to use the !important keyword in your CSS rules. This sill ensure the default rules are overwritten.

12.3 Creating A New Skin or Altering The CSS (Advanced)


If you are intending to create a new skin for the template or if you wish to alter some of the existing CSS styles, a few things should be noted. For the purposed of page load speed the Mentor template uses compressed CSS stylesheets and Javascript. This means that instead of individually loading a separate stylesheet for different elements, the CSS stylesheets have been compressed into one file which is loaded by Mentor hence saving time with only one HTTP request. The consequences of this however is that if any changes are made to the uncompressed stylesheets, these will not be reflected in the compressed version without a re-compression being done. Therefore if you wish to change the css stylesheets you have 2 options.

  1. Create a new stylesheet and insert it after the minimised CSS stylesheet ensuring to use the !important keyword after each rule you wish to overwrite (See below)
  2. Replace the minified CSS styles with the uncompressed versions. These can be found in the file entitled uncompressed.html which sits in the root folder with all of the other HTML files. Inside this file you will see that all of the CSS (and JavaScript) import statements are shown there individually. To successfully swap the compressed CSS for the uncompressed CSS in the template, all of these import statements will need to be pasted into each HTML page (the head section), replacing the single minified import statement. Once this is done, you may insert new CSS styles directly into these CSS files and the changes will be reflected in your page when you refresh.
New Stylesheet
<!-- compressed stylesheets -->
<link rel="stylesheet" href="css/Mentor.min.css" />
<!--new stylesheet-->
<link rel="stylesheet" href="css/new_style.css" />
Uncompressed Stylesheets
<!-- uncompressed stylesheets -->
<link href="js/pe.volo/themes/common.css" rel="stylesheet" />
<link href="js/pe.volo/themes/default/style.css" rel="stylesheet" />
<link href="js/pe.flare/video/theme/style.css" rel="stylesheet" />
<link href="js/pe.flare/themes/common.css" rel="stylesheet" />
<link href="js/pe.flare/themes/default/skin.css" rel="stylesheet" />
<link href="css/prettify.css" rel="stylesheet" />
<link href="css/bootstrap.css" rel="stylesheet" />
<link href="css/bootstrap-responsive.css" rel="stylesheet" />
<link href="css/menu_layout.css" rel="stylesheet" />
<link href="css/menu_skin.css" rel="stylesheet" />
<link href="css/styles.css" rel="stylesheet" />
<link href="css/slider_ui.css" rel="stylesheet"/>
The uncompressed CSS stylesheet and JavaScript import statements can be found inside the head section (CSS) and the bottom of the body section (JS) of the HTML file named uncompressed.html, found in the root download package for convenience sake.

13.2 Rollover Effects on iDevices


Some of the interesting effects used in Mentor are seen or experienced by the user only when rolling over an element using their mouse. However when using a touch screen device such as the iDevices, the rollover state does not feature as prominently. It is still possible to see rollover effects as follows. In Apple's mobile Safari browser the way to activate a rollover state for an element is by Soft Clicking. This is simply a normal click, but as long as the element being clicked has a rollover state, then this state will be activated and a second click is then needed in order to visit the link or page or whatever is the default action of the link itself.

Sometimes however, rollover effects have been intentionally switched off in the CSS because they will not function correctly on a touch based device due to the different nature of a touch user interface from that of a mouse based one.



13.3 Mentor and Android


Mentor has been tested on the Android operating system using a Samsung Galaxy Tab 10.1. running the Honeycomb version of the OS and also a Samsung Galaxy SII phone running the Ice Cream Sandwich version of the OS. While Mentor is working fine on the Honeycomb device, the template has been optimized for the latest version of the Android OS which is Ice Cream Sandwich. There are some well known bugs in the Android OS relating to video playback, there feature more so in Honeycomb, but they are a flaw in the default browser of the OS and are not related to the template. Additionally it should be noted that despite having been optimized for Android ICS, this does not mean that this template will work flawlessly on all Android devices. The reasons for this are as follows:

  1. There are just a vast number of different devices from different manufacturers, out there all running android and all with varying hardware specs. Each manufacturer has also tweaked or skinned the OS in their own way. Consequently it is impossible to perform testing on all such devices and to allow for any slight differences there may be in the way each device renders content.
  2. For reasons unknown to us, manufacturers of Android handsets and tablets are extremely slow in deploying updated to the OS. Consequently there are vast numbers of devices out in the wild which are running an array of different versions of the Android OS. Many of these devices may not have the hardware or software to run the template properly and so its impossible for us to promise full compatibility.
It is also worth noting that Android users may not enjoy as good an experience browsing this template as iOS users will. This is because at the time of writing the Android operating system does not allow for the experiencing of rollover states as iOS devices do.
